# 密钥

一个会话记录了智能体读取和运行过什么，因此它可能携带智能体碰巧看到的一个密钥：它打开的一个 `.env`、它打印出的一个
令牌。agit 在提交或推送任何东西之前扫描密钥，因此转录记录中的一个密钥不会意外地进入存储库或流向团队。

## 扫描与关卡

agit 在自己的入口点、在把控制权交给 git 之前，就地进行进程内扫描：

- `agit a commit` 扫描已暂存的索引（提交将写入的内容）。它先暂存提交的输入，因此一次 `agit a commit -a` 或一次
  pathspec 提交，会被完全按其将被写入的样子扫描。
- `agit a push` 扫描即将发布的提交范围。
- 来自 `agit a snap`、`agit watch`，或一次已完成的 `agit a merge` 的每一次快照，都以同样方式被扫描。

只有一次干净的扫描才让提交、推送或快照放行。`agit init` 还会在存储库上安装 pre-commit 与 pre-push 钩子，因此一次
裸 `git commit` 或 `git push` 也会被扫描。

agit 自己的提交和推送无法用 `git --no-verify` 蒙混放行：它们先在包装层扫描，然后向它们发起的 git 调用传入
`--no-verify`，因此即便 git 自身的钩子被跳过，包装层的扫描也总会运行。

随时手动扫描智能体的会话：

```bash
agit a scan
agit a scan --staged      # 扫描已暂存的索引，而非工作树
```

一个 agit 不识别的短横线标志会被拒绝，绝不会被当作扫描目标，因此 `agit a scan --no-verify` 无法默默地扫描一个不存在
的文件并报告「未发现密钥」。

## 当扫描发现问题时

扫描会打印每一个可疑密钥（文件、行、规则和一段摘录）并拒绝。一次被阻断的提交不会创建任何提交；一次被阻断的推送不会
发送任何东西。你有三条前进的路。

### 移除密钥

把密钥从会话中取出，然后重试。

### 标记误报

如果一处命中是已知安全的（一个文档示例、一个占位符），豁免它：

- 在那一行上加一条 `agit:allow-secret` 编译指示（pragma）。该指示豁免一个物理行，并随提交一同流转。
- 或把该字符串加入存储库的 `.agit-allow-secrets` 允许列表文件，它会豁免一个被匹配到的字符串。

中枢对两者都予以尊重，因此以此方式标记的一处误报，会同时通过本地扫描和服务器扫描。

### 覆盖本地扫描

```bash
AGIT_ALLOW_SECRETS=1 agit a push
```

`AGIT_ALLOW_SECRETS`（也可为 `true` 或 `yes`）让该操作在存在发现项的情况下仍然通过。它是一个可见、可审计的覆盖：
agit 每次予以尊重时都会披露它，指明操作与发现项。这正是它与 `git --no-verify` 的区别，后者不留痕迹。该覆盖只清除
agit 的本地扫描。一次推送到中枢仍会遭遇中枢自己的扫描，而该标志触及不到它。

## 存储库作用域对环境作用域

扫描运行在**智能体存储库（Agent Store）**上，即位于 `$AGIT_HOME` 之下、存放转录记录的 git 仓库。那正是密钥最终去向
之处，因为一份转录记录正是记录智能体所读令牌的载体。提交、推送、合并这些关卡，以及 `.agit-allow-secrets` 允许列表，
都存在于存储库上。你的**环境（Environment）**（代码仓库）自有其密钥纪律；agit 不扫描它，而在那里运行普通 `agit
<git-args>` 就是原封不动的 git。

## 清除已提交的密钥

如果一个密钥在你察觉之前就进了历史，`agit a purge-history` 会重写存储库，使任何提交中都不残留明文。它带护栏：它检查
前置条件、要求一棵干净的工作树、在可用时使用 `git filter-repo`（否则回退到 `git filter-branch`），且从不自动推送。
它会打印你审阅重写之后应运行的确切强制推送命令。另见[加密](./encryption.md)，它使用同一命令来清除加密前的明文。

## 中枢会再次扫描

当你推送到中枢时，它会在入站途中于服务端扫描，并拒绝一次携带密钥的推送。这道后盾即便面对一个从未运行本地扫描的
客户端、或一个用 `AGIT_ALLOW_SECRETS` 清除了扫描的客户端也依然生效，因为那个标志是客户端侧的，不会流转到服务器。
中枢一侧参见[报告问题](../hub/reporting-problems.md)。

密钥扫描识别已知的密钥形态。它是一道强有力的后盾，但不是对某个新奇密钥的保证。

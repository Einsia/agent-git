# 在其他目录中运行的会话

agit 为一个会话所处理的仓库捕获它。你并不需要从仓库根目录启动智能体：agit 依据会话的运行位置以及它触及的文件来归属它。

## 自动捕获的情形

`agit a snap` 与 `agit watch` 会自动捕获以下会话，无需任何额外步骤：

- **在仓库内运行。** 工作目录为仓库根目录或其某个子目录的会话（`cd frontend && claude`）会被捕获，其方式与 `git`
  通过向上查找 `.git` 来定位仓库相同。无论该会话是编辑了文件还是仅读取了文件，都是如此。
- **在此处编辑过文件。** 从仓库上层或外部（某个父目录或工作区目录）启动、且编辑了本仓库内某个文件的会话会被捕获，
  因为它确实在此处工作过。agit 会报告它纳入了哪些会话、以及来自何处。

一个编辑了两个仓库中文件的会话同时归属于两者，并在你于各自仓库中执行 snap 时被分别捕获。每个自动捕获的会话都会经过
与普通捕获相同的密钥扫描。

## agit 会征询的情形

有一种情形确实存在歧义：从仓库**上层**启动、且仅**读取**了此处文件而未做任何编辑的会话。它无法与一次途经工作区、
只是顺带瞥了本仓库一眼的评审区分开来，因此 agit 绝不会默默地认领它。

- 在交互式终端中，`snap` 与 `watch` 会按目录各征询一次：`N session(s) ran in ~/dev and read files here but
  edited none. Capture for this repo? [y/N]`。
- 在没有终端的情况下（脚本，或 `watch` 守护进程），agit 会打印一条指向 `agit relocate` 的提示，并不捕获任何有歧义的会话。

## 迁移

`agit relocate` 是手动控制手段。它会列出每一个运行在别的目录、却大概率属于本仓库的会话（涵盖上述每种情形，包括只读的
会话），并在确认后将它们迁入：

```bash
agit relocate
```

```
Sessions that ran elsewhere but belong in /home/you/code/web:
  claude-code · /home/you/code · 2 hours ago
      "add a rate limiter"
bring these 1 session(s) into /home/you/code/web? [Y/n]
```

在以下场景中使用它：只读且位于上层的情形、批量迁移，或从某个无关目录启动却触及了本仓库的会话（这是自动捕获不会扫描的
唯一情形）。

| 标志 | 效果 |
|---|---|
| `<session>` | 迁移一个会话，按 id、转录记录路径，或其所记录目录的一个子串匹配。 |
| `--to <path>` | 覆盖目的地。它必须是一棵 git 工作树；会话会安装到一个由它派生的 slug 之下。 |
| `--yes`（`-y`） | 跳过确认。 |

目的地默认为本仓库的根。迁移会把历史移入存储库，因此它从不无人值守地进行：一个没有 `--yes` 的非交互 shell 会被当作
「否」。

## 相关

`agit resume --relocate` 处理相邻的情形：当一个已记录会话所对应的是同一项目被移动后的形态时，针对当前检出继续它，
并将会话所记录的路径重写到该检出之上。参见[恢复会话](./resuming.md)。

# 诊断

两条命令会报告安装状态。`agit doctor` 是一次快速的健康检查，可粘贴进 bug 报告；`agit debug` 写出一份完整的、经脱敏
的诊断包供你附上。两者收集相同的状态，且都会在写入或打印任何内容之前对密钥脱敏。不上传任何内容。

## 健康检查

```bash
agit doctor
```

`agit doctor` 打印一份人类可读的健康检查与环境摘要：`PATH` 上的运行时、你的 git 身份、活动存储库相对其远端的状态、
监视守护进程的状态，诸如此类。它不接受任何参数。当出现异常时先运行它，并在报告问题时粘贴它的输出。

## 诊断包

```bash
agit debug
```

`agit debug` 写出完整的诊断包：分节的文本，加上一份机器可读的 `debug.json`，顶部附有 doctor 摘要。它会回显该诊断包，
以便你在分享之前精确检视它收集了什么。

| 标志 | 效果 |
|---|---|
| `--out <dir>` | 将诊断包写入指定目录。 |
| `--rerun "<subcmd>"` | 在跟踪（trace）模式下重跑一条失败的 agit 子命令，并将其捕获进诊断包。 |

```bash
agit debug --out ./agit-debug
agit debug --rerun "a push"
```

## 收集哪些内容，以及它已被脱敏

诊断包收集 agit 所见的运行时及其会话转储、你的 git 配置与远端、存储库的状态及其远端、每一个 `AGIT_*` 环境变量，
以及解析出的活动智能体。它不收集会话转录记录的内容。

一切在写入之前都会经过脱敏处理。运行两层：

- 源头处的字段级脱敏：远端 URL 会被剥除其凭据，且每个 `AGIT_*` 变量的值都被无条件掩码（名称保留，值显示为
  `[redacted]`），因为一个值可能携带令牌。
- 对整个组装完成的诊断包做最终一遍：每个带协议方案的 URL 都被剥除其凭据，且密钥扫描器标记的任何行都被替换为脱敏标记。

由于诊断包已被脱敏并回显，你可以在附上它之前先读一遍。关于发往何处，参见[报告问题](../hub/reporting-problems.md)。

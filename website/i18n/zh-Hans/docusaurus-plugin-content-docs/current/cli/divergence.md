# 分叉

智能体记忆会分叉，然后重新汇合。子智能体（subagent）、一次 `/branch`，以及一次分叉（fork），本质上是同一操作在三种
作用域下的呈现，区别只在于分叉传播多远，以及它是否赢得自己的身份。`agit a log` 将结果渲染为一张 DAG，而 `agit a
merge` 是通用的重新汇合方式。

## 模型

| | 分叉的作用域 | 保留 aid？ | 生命周期 |
|---|---|---|---|
| **子智能体** | 在一个会话之内 | 是 | 短暂（返回结果时即消亡） |
| **分支（`/branch`）** | 在一个智能体之内：一个共享前缀的兄弟会话 | 是 | 持久 |
| **分叉（Fork）** | 跨智能体 | 否（一个新 aid） | 持久 |

- **子智能体**是一个自动重新汇合的分叉点。转录记录留存于运行时写入之处，在父会话之下；agit 记录该分叉点（哪一父轮次
  派生了它，以及它返回进哪一轮次），使其在 DAG 中可寻址。
- **分支**是一个与其父共享前缀、随后分叉的兄弟会话。这正是 Claude Code 的 `/branch` 所产生的。agit 记录父会话 id 和
  最后一个共享轮次。
- **分叉（fork）**是一个其会话被安装到新 aid 之下的分支（一次克隆加上 `agit a rebind --new-id`）。它保留自己的身份
  与自己的历史。

## 捕获，而非创建

默认路径是捕获。当你在运行时中执行 `/branch` 时，运行时写出一个共享前缀的新会话；agit 检测到该共享前缀并将分叉点
记录进该会话已提交的附属文件（sidecar）中，于是 `agit a log` 能渲染它、`agit a merge` 能调和它。存储库中没有 git
分支，也没有 `refs/heads` 的滋生：存储库保持单线，而 DAG 是一层逻辑叠加，以血缘（lineage）元数据的形式挂在每个会话上。

## `agit a log` 中的 DAG

```bash
agit a log
```

`agit a log` 将存储库的会话渲染为一棵终端树。根（没有显示父的会话）按新近程度顺序先打印，而每个 `/branch` 子会话
缩进悬挂在它所共享前缀的父之下。子会话显示它从父处何处分叉（`branched at <id>`），并显示它分叉后的尖端提示，而非它
与父共享的开头，这样起头相同的兄弟会话仍能被区分开来。派生过子智能体的会话会被标注（例如 `+2 subagents`）。每个
节点都带一个短会话 id。

传入 `--raw`（或 `--git`）可得到存储库上逐字节的 `git log`，而非会话视图。

## 重新汇合

`agit a merge` 是重新汇合，它以 aid 为键：

- **分支**用 `agit a merge <session-or-ref>` 重新汇合。aid 相同，因此历史融合，合并后的会话成为该智能体的最新会话。
- **分叉（fork）**用 `agit a merge <agent>` 重新汇合。aid 不同，因此二者仅通过对话方式调和，各自保留自己的历史。
- **子智能体**在运行时内返回结果时就地重新汇合；agit 只是记录它。

`/branch` 所产生的共享前缀结构，正是拼接（splice）合并据以调和的对象。关于一次调和如何运行，参见[合并会话](./merging.md)；
关于 aid，参见[核心概念](../get-started/concepts.md)。

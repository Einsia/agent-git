# 阅读会话

智能体页面会列出它的会话。打开其中一个即可阅读。会话视图正是中枢的意义所在：它把一份记录（transcript）转化为一个人可以阅读并加以判断的东西，保留了推理过程，而不仅仅是差异（diff）。

## 对话

视图的主体是有序的对话：用户与助手的轮次按其发生的顺序交错排列，而不是被压平成两份彼此独立的列表。每一轮的文本都以 markdown 渲染，且 markdown 在显示前会被净化（sanitize），因此一份记录无法向页面注入活动内容。

每一轮显示：

- 它的**角色**（用户或助手），
- 它的**文本**，以 markdown 渲染，以及
- 该轮所调用工具的**工具调用计数**，让你无需翻阅原始的工具载荷即可看出智能体在何处触及了环境。

极长的会话在视图中会被截断；当对话被截断时，页面会加以标记，而完整历史始终可以通过克隆（clone）智能体的存储库获得。

## 脊线

在对话之外，视图还展示会话的脊线（spine）：一份紧凑、有序的事件序列摘要（提示、回复、工具调用、编辑）。它由与对话相同的事件遍历构建而成，因此在你阅读文字之前，就能一眼读出一个会话的形状。

## 来源认证徽章

视图携带一个来源认证（provenance）徽章，即关于谁产出了该会话以及它是否完整的密码学裁决。它会读出以下之一：

- **verified as `<person>`**：签名核验通过，且提交者邮箱映射到一个持有匹配设备密钥的已注册账户。
- **verified**：签名与内容完整、密钥已知，但未归属到某个中枢账户。
- **signed, unregistered**：已签名且完整，但签名密钥未在中枢登记。
- **unsigned**：无签名。这不是错误；来源认证会优雅降级。
- **key mismatch**：提交者邮箱声称属于某个账户，但签名密钥并非该账户的任何一个密钥。这可能是一次伪造，绝不会被显示为已验证。
- **content tampered** 或 **bad signature**：记录与被签名的内容不匹配。

要让你自己的会话读作 `verified as you`，请登记一个设备密钥并验证你的邮箱。参见 [签名密钥](./signing-keys.md) 与 [验证谁产出了会话](../integration/provenance.md)。

## 文件与来源认证事实

视图还会展示会话从其提交中提取的事实：运行时（runtime）、模型、分支、工作目录、作者与时间，外加该会话所触及的**文件**。在决定一个会话是否值得拉取或合并时，这些正是你要权衡的内容。

## 修订版本

会话随着被捕获与重新推送而累积修订版本。视图会列出它们（每个都带有其提交、时间与主题），并允许你把记录固定到某个特定修订版本，以便按会话在那个时点的样子来阅读它。来源认证裁决是在你正在阅读的那个修订版本上计算的，因此徽章始终描述屏幕上的字节。

## 比较两个修订版本

从智能体视图中，你可以比较其历史中的两个时点。这种比较是语义化的：它展示两个修订版本之间新增和移除的指令、文件与结论，而非原始的记录字节。这是在本地调和已分叉工作的只读对应物；参见 [分叉](../cli/divergence.md) 与 [调和已分叉的会话](../cli/merging.md)。

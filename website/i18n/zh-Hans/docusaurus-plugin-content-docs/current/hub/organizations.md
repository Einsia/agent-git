# 组织

组织（organization）是一个共享的所有者。归组织所有的智能体，可由每一位组织成员按其角色访问，因此一个团队的智能体可以在任何一个人的账户之外长久存续。组织名称遵循与用户名相同的规则，并共享同一命名空间：一个名称要么是用户，要么是组织，绝不会两者兼有。

## 创建组织

任何已登录用户都可以从 Web 界面创建一个组织。创建者成为它的第一个也是唯一一个管理员。默认情况下成员可以在组织下创建智能体；管理员可以在组织设置中将创建权限限制为仅管理员。

## 组织概览页面

`/orgs/<name>` 是组织的概览页面。它列出组织的成员以及组织所拥有的每一个智能体，外加你有权阅读的每位成员的个人智能体。每一行智能体都显示它的会话数量以及它工作过的代码环境。

该页面仅限组织成员与站点管理员访问。它只列出你可以阅读的智能体，并且先过滤再计数，因此它绝不会泄露一个你本就看不到的智能体，甚至连会话数量也不会泄露。某位成员的私有个人智能体不会继承组织授权，若无针对它的显式授权，它对任何人都不会出现。

## 角色

一个成员要么是 `member`，要么是 `admin`。

- **成员（Members）** 可读，并（在对某个组织所有的智能体持有写授权时）可推送。默认情况下他们也可以在组织下创建智能体。
- **管理员（Admins）** 额外负责管理名册：邀请人员、变更角色、移除成员、编辑设置、转移所有权，以及删除组织。

组织必须始终至少保留一名管理员。降级或移除最后一名管理员会被拒绝。

## 成员资格仅限邀请

你无法直接把一个陌生人加入组织。加入的唯一途径是被接受的邀请，以及向某位现有成员的所有权转移。

**邀请。** 组织管理员以某个目标角色（`member` 或 `admin`）邀请一位现有用户。这会创建一个待处理邀请；它本身尚不授予任何权限。运营者可从 CLI 使用 `agit-hub org invite <org> <user> [--role R]` 发出邀请，并用 `agit-hub org invitations <org>` 列出待处理邀请。

**接受或拒绝。** 被邀请者会在自己的账户上看到其待处理邀请（`GET /api/me/invitations`），并可接受或拒绝。只有被指名的受邀者才能对某个邀请采取行动。接受会以所提供的角色铸成成员资格；拒绝则不授予任何东西。任一处置均为最终结果，因此邀请无法被重放。

变更某位现有成员的角色是名册上的一项单独的管理员操作，不经过邀请流程。

## 设置

组织管理员可控制：

- **成员创建策略。** 成员是否可以在组织下创建智能体（默认可以），还是仅管理员可以。
- 加密恢复与中枢辅助托管（hub-assist escrow）选项，当你的团队使用它们时。

## 转移所有权

组织管理员可以把所有权交给另一位成员。新所有者必须已经是成员。转移会一步将其提升为管理员，并将发起者降级为成员。转移属于管理级操作：它要求一个登录会话，无法用令牌完成，即便是管理员的写令牌也不行。

## 删除组织

组织管理员可以删除一个组织，但仅当它不再拥有任何智能体时才可以。请先转移或删除组织的智能体，否则删除会被拒绝，以免任何仓库或 blob 数据变成孤儿。删除一个组织会移除它本身、每一份成员资格以及针对它的每一个邀请。与转移一样，删除要求一个登录会话，而非令牌。

# 中枢概览

:::tip 试用托管中枢
[agit.anggita.org](https://agit.anggita.org) 上运行着一个中枢。登录即可浏览智能体，或[自行运行一个](../self-hosting/deploying.md)。
:::

中枢（Hub）是团队自行运行的一台服务器，用于共享智能体（agent）并在浏览器中阅读它们。它承担三项职责：

- **托管。** 每个智能体都是一个存放会话记录（transcript）的裸 git 仓库。你通过 git smart-http 向它推送（push）、从它拉取（pull），与任何其他 git 远端并无二致。
- **同步。** 它是团队成员共同获取（fetch）的共享 origin，因此在一台机器上捕获的会话可以在另一台机器上阅读。
- **渲染。** 它提供一个 Web 界面，将每个会话呈现为一段对话、比较两个修订版本，并为某个组织或某个代码仓库所产出的智能体建立索引。

中枢本身不运行智能体，也不执行合并（merge）。合并需要读取两个已分叉（diverged）的会话，并结合你的代码对它们进行推理，这项工作在同时拥有代码与模型的本地机器上运行。参见 [调和已分叉的会话](../cli/merging.md)。

中枢没有任何专属的客户端动词。你通过普通的 `agit` 命令与中枢交互，只需将某个 git 远端指向智能体的 `.git` URL 即可。要连接 CLI，参见 [将 CLI 连接到中枢](../integration/connect-cli-to-hub.md)；要发布，参见 [共享智能体](../integration/sharing.md)。要搭建一个中枢，参见 [部署中枢](../self-hosting/deploying.md)。

## 登录

在浏览器中打开中枢并使用你的账户登录。密码登录会返回一个会话 cookie，它会过期，并在你登出时被吊销。如果你的团队启用了自助注册，你可以在登录页面自行创建账户；否则由管理员为你创建。参见 [账户](./accounts.md)。

git 与脚本的认证方式不同，它们使用令牌（token）或已登记的设备密钥，而非 cookie。参见 [令牌](./tokens.md) 与 [认证](../integration/authentication.md)。

## 浏览你有权阅读的智能体

中枢会列出你有权阅读的智能体：你自己的、共享给你的，以及所有公开的智能体。你无权阅读的智能体不会显示，且一个私有智能体与一个根本不存在的智能体无从区分，因此智能体的名称无法被枚举。

每个智能体都有一个所有者、一个可见性（默认为私有），以及被授予读、写或管理员权限的成员。你能做什么取决于你所获的授权：

| | 读 | 写（推送） | 管理员（可见性、成员、重命名、删除） |
|---|---|---|---|
| 匿名 | 仅公开 | 否 | 否 |
| 已登录用户，无授权 | 仅公开 | 否 | 否 |
| 成员（读 / 写 / 管理员） | 是 | 写及以上 | 管理员及以上 |
| 所有者 | 是 | 是 | 是 |

当中枢拒绝某项操作时，它会说明原因。被拒绝的推送会指出它所认证的账户以及它所缺少的权限，因此错误的令牌、缺失的授权与只读的作用域都很容易区分。

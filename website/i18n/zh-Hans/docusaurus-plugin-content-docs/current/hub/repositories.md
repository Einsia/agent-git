# 代码仓库

`/repos` 是代码仓库索引。如果说智能体列表回答的是“有哪些智能体存在”，那么代码仓库索引则将其反转，回答的是“智能体们在哪些代码上工作过，以及它们分别是哪些智能体”。

智能体捕获的每一个会话都会记录它所运行于的代码环境。索引按该环境对会话进行分组，因此每一行都是一个代码仓库，并附上曾触及它的智能体。一个仓库常常由多个智能体共同工作；它只列出一次，而每个附上的智能体都会显示它在该环境中的会话数量。关于环境这一概念，参见 [核心概念](../get-started/concepts.md)。

## 一行显示了什么

每一行代码仓库都携带：

- 在该环境中拥有会话的智能体，每个都带有其会话数量，
- 这些智能体的总会话数量，以及
- 该环境最近一次被触及的时间，取自其中最新的会话提交。

从某一行打开一个智能体，会带你进入该智能体，你可以在那里阅读它的会话。参见 [阅读会话](./reading-a-session.md)。

## 它不会揭示什么

索引仅由你有权阅读的智能体构建而成。你无权阅读的智能体不会为任何一行贡献任何东西，其存在也无法从计数中被推断出来。一个仅由你看不到的智能体所工作过的仓库根本不会出现。管辖智能体列表的那套逐智能体的读权限检查，同样管辖这里的每一个计数。

在旧布局下捕获的会话不带任何环境标识（slug），无法为某一行代码仓库充当键，因此被从本索引中略去。

# 账户

你的中枢账户是 Web 界面识别你身份的凭据。它承载一个密码登录、一个可选的已登记邮箱、一个可选的第二因子，以及任意数量的设备签名密钥。本页介绍如何获得账户、如何登录，以及账户页面上的自助安全控制项。

## 获得账户

账户有两种创建方式，具体适用哪一种取决于中枢的部署方式。

- **自助注册**，当运营者启用它时。登录页面会提供一个“创建账户”表单。它会创建一个普通的、非管理员账户并将你登录。注册永远无法授予管理员权限。
- **管理员创建**，其他情况下。中枢默认仅限邀请，由管理员运行 `agit-hub user add <name>` 来创建你的账户并把密码交给你。

用户名为 2 至 32 个小写字符，取自 `[a-z0-9._-]`，且不得以点开头。用户名与组织名称永远不能是同一个字符串。

关于运营者如何开启注册，参见 [部署中枢](../self-hosting/deploying.md)。

## 登录与登出

使用你的用户名和密码登录。登录成功会返回一个会话 cookie；该 cookie 是每一次 Web 操作的凭据。登出会吊销该会话。

密码提示被刻意设计得较慢（中枢使用 argon2id 进行哈希），并且不会告知登录失败是因为用户名错误还是密码错误。如果你的账户已被管理员禁用，即便密码正确也只会返回一条清晰的“账户已被禁用”消息，而非一个会话。

令牌与设备密钥用于 git 和脚本，而非 Web 界面；诸如签发令牌之类的管理员操作始终要求一个登录会话，绝不接受令牌。参见 [令牌](./tokens.md)。

## 更改密码

在账户页面，通过提供你的当前密码和一个新密码（至少 8 个字符）来更改密码。更改密码会吊销该账户的所有其他会话，但会保持你执行更改的那个标签页处于登录状态，因此轮换密码可以踢掉被窃取的 cookie，而不会把你自己登出。

## 重置遗忘的密码

如果你被锁在门外，可在登录页面请求重置。中枢会生成一个一次性、会过期的重置链接。默认由运营者转发投递：该链接会被写入服务器日志，供管理员转发给你。打开链接并设置一个新密码。使用该链接会吊销该账户的所有会话。管理员也可以直接重置被锁定账户的密码。

无论账户是否存在，请求端点都以相同方式应答，因此它绝不会泄露哪些用户名已被注册。

## 验证你的邮箱

你账户的记录邮箱，是你主设备密钥上的提交者（committer）邮箱，在你登记密钥时自行声明（参见 [签名密钥](./signing-keys.md)）。验证它是抢注防护的关口：只有当你证明了对该地址的控制权后，会话的提交者邮箱才会归属到你的账户（即 `VERIFIED AS` 的来源认证裁决）。

- 账户页面会在你的邮箱旁显示“已验证”或“未验证”徽章。
- 使用 **Resend** 生成一个新的验证链接。与密码重置一样，投递由运营者转发（URL 会被记入日志供管理员转发），令牌绝不会在页面中返回。
- 打开链接以将该地址标记为已验证。该链接仅可使用一次，并在 24 小时后过期。
- 更改已登记的邮箱（通过登记一个带有不同地址的密钥）会重置已验证标志，因此新地址必须重新证明。

管理员也可以通过 `agit-hub user verify-email <name>` 带外强制验证某个地址。链接的基址来自运营者的 `AGIT_HUB_BASE_URL`；参见 [中枢配置](../self-hosting/configuration.md)。

## 双因子认证（TOTP）

在账户页面开启第二因子。一旦启用，仅有正确的密码将不再能登录你；登录还需要一个当前有效的验证码或一个未使用过的备份码。

1. **登记。** 中枢生成一个密钥，并将其显示为一个 `otpauth://` 配置 URI（以及原始密钥），供认证器应用使用。登记本身尚不会开启 2FA。
2. **确认。** 输入应用中的一个 6 位验证码。成功后 2FA 即启用，中枢会返回 10 个一次性备份码。它们只显示一次；只有其摘要会被存储。请把它们保存在安全的地方。
3. **登录。** 从此之后，登录会要求输入验证码。一个当前有效的 TOTP 或任意一个未使用过的备份码均可。
4. **停用。** 使用当前有效的验证码、未使用过的备份码或你的账户密码中的任意一项来关闭 2FA。

在 2FA 已启用的情况下重新登记会被拒绝；请先停用它。如果你同时丢失了认证器和备份码，管理员可以清除你的 2FA，以便你重新登记。

# 签名密钥

签名密钥（signing key）是让你的会话归属于你的设备密钥。每台机器持有一个 ed25519 密钥，其私钥部分永不离开该机器；中枢只存储公钥部分。一旦你的密钥被登记且邮箱被验证，你所签名的会话在中枢上就会读作 `verified as you`。这是向代码托管平台添加 SSH 密钥这一做法的 HTTPS 版本：一次 Web 登录即可登记密钥，从此之后机器便凭该密钥自证身份。

## 登记密钥

登记在客户端离线完成，再粘贴进 Web 界面，因此没有任何私钥或令牌穿越网络。

1. 在机器上，为你的中枢账户打印一个登记块：

   ```bash
   agit identity register <you> [--label <name>]
   ```

   这不发起任何网络调用。它会打印一个签名块，其中只承载公钥材料：你的 ed25519 与 X25519 公钥、一个自行声明的提交者邮箱、一个可选的设备标签（默认为你的主机名），以及一个证明该机器持有匹配私钥的签名。

2. 在中枢中，打开 **Account -> Signing keys -> Add a signing key** 并粘贴该块。你已登录的会话为此次添加授权，正如在代码托管平台添加 SSH 密钥一样。该块是为某个特定账户签名的，因此为一个账户打印的块无法被粘贴到另一个账户之下。

该块编码了一个单调递增的纪元（epoch），因此重新运行 `agit identity register` 并粘贴较新的块会更新密钥，而不会被当作重放而拒绝。

关于客户端一侧的完整内容（轮换、展示与固定密钥），参见 [身份](../cli/identity.md)。

## 每个账户多个密钥

一个账户可持有任意数量的设备密钥，每台机器一个，各有其标签。在一台新机器上登记密钥会加入到集合中；它绝不会替换你已经登记的密钥。当一个会话的签名密钥匹配你已登记密钥中的任意一个时，它就归属于你，因此你可以在多台笔记本电脑与 CI 运行器之间保持同一身份。

从同一个 Signing keys 页面（或在持有某密钥的机器上使用 `agit identity revoke`）吊销一个你不再信任的密钥。被吊销的密钥会从集合中移除，不再归属任何东西。

## 登记解锁了什么

**归属。** 在密钥被登记且其邮箱被验证后，中枢会把一个已签名、完整的会话升级为 `verified as <you>`：签名核验通过、提交者邮箱映射到你的账户，且签名密钥是你的密钥之一。没有匹配的密钥时，同一个会话只会读作 `signed, unregistered`；而一个已签名的会话，若其邮箱声称属于你的账户但其密钥并非你的，则读作 `key mismatch`，绝不会是已验证。参见 [验证谁产出了会话](../integration/provenance.md) 与 [阅读会话](./reading-a-session.md)。

你在登记时声明的提交者邮箱，是从一个会话的 git 提交者通往你账户的桥梁，因此在归属信任它之前，它必须先被验证。更改邮箱会重新触发验证。参见 [账户](./accounts.md)。

**基于密钥的中枢认证。** 同一个已登记的密钥让 git 无需复制粘贴令牌即可向中枢认证：一次挑战-应答握手会即时从该密钥铸出一个短时效的 bearer 令牌，因此推送、拉取、获取与克隆全都凭该密钥认证。参见 [认证](../integration/authentication.md)。

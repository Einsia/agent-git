# agit

agit 为你的 AI 编码智能体（agent）所产生的原始会话（session）转录记录（transcript）提供版本控制。当你使用
Claude Code 或 Codex 工作时，每一次会话都包含了智能体读取、运行和修改的内容，以及一路引导至此的各条提示词与
回复。agit 会将这份转录记录保存进一个 git 仓库，于是一次会话便能像你的代码一样，接受版本化、恢复、共享与协调。

如今，一次智能体会话是用完即弃的。代码差异（diff）会留存在你的代码历史中，而产出它的推理过程却不会。agit 保留的
是推理过程，而不仅仅是差异。从一份保存下来的会话出发，你可以带着完整加载的上下文恢复（resume）该智能体，把这个
智能体连同它的历史移交给同事，并协调两个人从同一起点分叉（diverge）出的各自运行结果。

## 两个界面

agit 有两个界面，你会在不同的时机使用它们。

**CLI（`agit`）** 运行在你的机器上。它将会话记录进一个本地 git 仓库（也就是智能体的存储库），恢复会话，在
Claude Code 与 Codex 之间迁移会话，并向远端推送（push）和从远端拉取（pull）。凡是你能运行 `git` 的地方，都能
运行 `agit`。`agit <git command>` 会针对你的代码仓库运行 git，行为不变。在 `agit` 之后加上 `a`，同一条命令便会
针对智能体的存储库运行，因此 `agit log` 显示你的代码历史，而 `agit a log` 显示智能体的各次会话。参见
[CLI 概览](../cli/overview.md)。

**中枢（`agit-hub`）** 是一台你的团队向其推送的服务器。它存储各个智能体，用签名密钥（signing key）对推送与拉取
进行认证，并提供一个 Web 界面，让你浏览某个智能体的各次会话，并把转录记录以对话形式（而非原始 JSON）阅读。当有
一人以上在同一个智能体上工作时，或当你想在浏览器中阅读某次会话时，就使用它。参见
[中枢概览](../hub/overview.md)。你可以使用[托管的中枢](https://agit.anggita.org)，也可以
[自行运行一个](../self-hosting/deploying.md)。

## agit 解决什么问题

- **会话是一次性的。** 关掉终端，智能体的上下文就没了。agit 会记录每一次会话，让你日后能在上下文完好无损的情况下
  [恢复](../cli/resuming.md)它。
- **只有差异被共享。** 同事看到的是发生了什么改动，而不是智能体为什么这样改。将一个智能体推送到中枢，同事便能
  [克隆](../integration/sharing.md)其中的会话、提示词与工具活动。
- **并行运行会分叉。** 两个人恢复同一个智能体，各自产生新的会话。agit 会追踪这种
  [分叉](../cli/divergence.md)，并通过
  [对话式合并（dialogue merge）](../cli/merging.md)协调两侧。

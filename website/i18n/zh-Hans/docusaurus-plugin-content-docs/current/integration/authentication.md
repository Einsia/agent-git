# 认证

中枢（Hub）接受三类凭据，分别对应不同的调用方。

| 凭据 | 适用对象 | 授予的权限 |
|---|---|---|
| 基于密钥的认证（key-auth） | 你的各台机器（push、pull、fetch、clone） | 由已注册密钥签发的短时效令牌 |
| 令牌（token） | 脚本，或未注册密钥的机器 | 限定范围、会过期的持有者（bearer）凭据 |
| Cookie 会话 | 浏览器中的用户 | 完整账户权限，包含管理操作 |

## 基于密钥的认证（Key-auth）

agit 充当你的中枢的 git 凭据助手（credential helper）。当 git 需要某个中枢主机的凭据时，它会运行
`agit credential get`，后者执行一套“挑战、签名、交换”的握手流程，将一枚新签发的短时效令牌交回给 git。你无需
在每次 push 时复制任何内容。

握手流程如下：

1. `GET /api/auth/challenge` 返回一个一次性 nonce。
2. agit 使用本机的 ed25519 密钥签署一份断言，将该 nonce、你的用户名、本机公钥、作为受众（audience）的中枢，
   以及一个近期的过期时间绑定在一起。
3. `POST /api/auth/key` 依据账户已注册的密钥验证签名，并签发一枚持有者令牌，其时效以分钟计，而非以天计。

该令牌在中枢的 tokens 表中是一条普通记录，范围限定为写权限、归属于你的账户，并可在 Web UI 的令牌列表中查看。在
一次 push 中，git 可能会请求两次（先是 info/refs，再是 receive-pack）；agit 将该令牌以仅属主可读的方式缓存到磁
盘上，因此一次 push 只签名一次，而非每次请求都签名。

### 基于密钥的认证的作用范围

agit 仅为中枢主机接入凭据助手：`AGIT_HUB_URL` 的主机，以及当前活动智能体（agent）所绑定的存储库（store）远程的
主机。github、gitlab 等其他远程会原样交由 git 自身的助手处理。同一套接线机制覆盖 `push`、`pull`、`fetch` 和
`clone`。

`clone` 更为严格。clone URL 属于不受信任的输入，因此 agit 只有在该 URL 的主机已被本机声明为中枢时才会为其自动
签发令牌。任意的 https URL、github 或 gitlab 都不会触发签名挑战，因此你的账户用户名与公钥绝不会被发送到一个你尚
未信任的主机。请先声明中枢，可通过 `AGIT_HUB_URL` 或一次先前的 push 完成。

已签名的断言将中枢列为其受众，从而将断言绑定到某一个中枢，防止签名被重放到另一个中枢。在服务端，运维方通过
`AGIT_HUB_PUBLIC_URL` 固定（pin）该受众；参见[自托管配置](../self-hosting/configuration.md)。

该助手在设计上刻意宽容。对于任何非中枢主机、未知账户或握手过程中的任何错误，它不打印任何内容并以 0 退出，使 git
回退到其正常的 Basic 认证提示。push 绝不会因为基于密钥的认证不可用而硬性失败。

使用 [`agit identity register`](../cli/identity.md) 注册某台机器的密钥，并将输出的密钥块粘贴到 Web UI 中。
关于首次连接流程参见[连接 CLI](./connect-cli-to-hub.md)，关于管理已注册密钥参见[签名密钥](../hub/signing-keys.md)。

## 令牌（Tokens）

令牌是你在 Web UI 中创建的、限定范围且会过期的持有者凭据。可在脚本中使用，或用于未注册密钥的机器。你可以将其范围
限定到单个智能体、设定时限，并随时吊销。令牌是权限的上限，而非权限的来源：只读令牌依然只能读取。

可通过 git 的密码提示、存储库远程 URL 的密码字段，或环境变量来提供令牌。

| 变量 | 含义 |
|---|---|
| `AGIT_HUB_URL` | 中枢的地址；同时声明该主机，使基于密钥的认证与 clone 自动签发能够识别它 |
| `AGIT_HUB_TOKEN` | 持有者令牌；会覆盖从远程 URL 中解析出的任何凭据 |
| `AGIT_HUB_USER` | 用于认证的账户；会覆盖凭据助手所用的已注册账户 |

关于创建与吊销令牌，参见[令牌](../hub/tokens.md)。

## Cookie 会话

用户通过浏览器登录。密码登录会返回一个会话 cookie，它会过期，也会在登出时被吊销。管理操作——更改智能体的可见性、
管理成员、重命名、删除——都需要登录。它们绝不对令牌开放，即便是管理员的令牌也不例外。如果你的中枢启用了自助注册，你
可以自行创建账户；否则由管理员创建。参见[账户](../hub/accounts.md)。

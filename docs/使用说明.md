## 1. 它解决什么问题

代码通过 Git 对团队公开了,但 agent 的工作困在私人 session 里 —— 它读过什么、做过什么判断、
下一步是什么,别人看不到、没法复用。

agit 直接管理 agent 的**原始会话**:Claude Code 本来就把整个会话 dump 到磁盘,agit 把这坨
版本化、push/pull 同步;合并时让**另一个 agent 读懂对面的会话、自行合并**,只有真冲突才问你。

> 之前有过一版"手写 fact + 证据 + 确定性合并"的模型,已废弃。为什么、以及新旧取舍见
> [`风险分析.md`](风险分析.md)。

## 2. 核心:两个库 + agent 驱动的合并

```
你的项目/
├── models/user.ts            ← Environment:你的代码,agit 原封不动
├── .gitignore                ← 自动加入 .agit/
└── .agit/
    ├── agent/                ← Agent Store:独立 git 仓库
    │   ├── agent.toml
    │   └── sessions/claude-code/
    │       ├── <id>.jsonl         完整转录
    │       └── <id>/subagents,tool-results
    └── workspace/            ← WorkspaceRevision:Agent↔Environment 配对
```

### 两个 scope

```
agit <git 命令>       →  透明作用在你的代码仓库(Environment)
agit -a <git 命令>    →  同构作用在 Agent Store
```

> **⚠️ 歧义(务必记住)**:`agit -a commit`(agent 库)vs `agit commit -a`(代码库,`-a` 是 git
> commit 的参数)。scope 开关只认紧跟 agit 的第一个 token。

Agent Store 就是 git 仓库,所以 `push` / `pull` / `clone` / `fetch` / `log` / `diff` 全部透传白拿。

## 3. 安装

```bash
./build.sh --release              # 为什么不是 cargo build:见根目录 README
cp target/release/agit ~/.local/bin/

cd 你的项目
agit init                         # 建 Agent Store;clone 后需重跑
```

> **clone 后要重跑 `agit init`。** `.gitignore`/hooks 的道理:hook 不跟着 clone 走。

## 4. 五分钟上手

```bash
cd 你的项目 && agit init
agit -a sync                                    # 镜像本项目的 Claude session dump
agit -a add -A && agit -a commit -m '记录会话'
agit -a push                                    # 发布(push 前扫密钥)

# 同事那边
agit clone <url> && agit -a fetch origin
agit -a reconcile origin/main                   # agent 合并,真冲突才问你
```

## 5. 命令详解

### 原生动词

| 命令 | 作用 |
|---|---|
| `agit init` | 建 Agent Store + 密钥 hook |
| `agit -a sync [--from <rt>]` | 把 runtime 的 session dump 镜像进 Agent Store(默认 claude-code;`sync codex` 位置参数即 runtime 简写) |
| `agit -a reconcile <ref>` | 并入 `<ref>` 的 session,让 agent 合成统一上下文写进 CLAUDE.md,真冲突才问你 |
| `agit -a reconcile <ref> --dry-run` | 只预览合成结果,不合并、不写盘 |
| `agit -a reconcile --continue` / `--abort` | 文本冲突手动解决后定稿 / 放弃进行中的合并 |
| `agit convert <src> --to <rt> [--write]` | 把一份 session 转成另一个 runtime 能 resume 的会话(默认 dry-run;`--write` 落盘并给 resume 命令) |
| `agit clone <url>` | 一条命令拉团队 Agent Store(clone + 装 hook) |
| `agit -a scan [--staged]` | 扫 session dump 里的密钥 |
| `agit workspace [log]` | 看 Agent↔Environment 配对 |
| `agit workspace restore [N\|<agent-rev>]` | 把两个库一起退回某条配对记录的联合状态(JointVersionControl 的撤销) |
| `agit adapter` | 列出 runtime adapter(claude-code、codex,均已实现) |

### 透传(和 git 完全一致)

`add` / `commit` / `status` / `log` / `diff` / `branch` / `checkout` / `merge` / `push` /
`pull` / `fetch` / `clone` …… 不加前缀作用在代码仓库,加 `-a` 作用在 Agent Store。

`agit` **不替代** `git`,两者并存。

## 6. reconcile:合并是怎么发生的

会话是不同 uuid 的文件,所以 `git pull/fetch` 拿到对面的会话时**不会文本冲突**,它们只是并排落进
`sessions/`。真正的"合并"是语义层的,由 `reconcile` 做:

```bash
agit -a fetch origin
agit -a reconcile origin/main
```

1. 算出 `<ref>` 带来的是哪些新 session,`git merge` 把它们并进来(文件层,通常无冲突)。
2. 把两边会话各自的**紧凑摘要**喂给一个 LLM(默认本机 claude),让它:
   - 读懂对面做了什么、什么风格/约定;
   - 合成一份**统一工作上下文**(目标、已知结论、约定、下一步),写进项目根 `CLAUDE.md` 的
     受管区块(下个 Claude Code 会话自动加载);
   - **只挑出真正矛盾**的点。
3. 没矛盾就直接合;有矛盾就停下、列给你(也落一份 `.agit/workspace/reconcile-conflicts.md`),让你改定 CLAUDE.md 或重跑。

**模式:**
- `--dry-run`:算好 incoming、跑一遍合成,只**打印**统一上下文与冲突,不 `git merge`、不写 CLAUDE.md。
- 文本冲突(非 session 文件)时**保留合并中状态**(不再 abort 掉再喊你手动处理):改好冲突文件、
  `agit -a add -A`,再 `agit -a reconcile --continue` 定稿;或 `agit -a reconcile --abort` 放弃。
- **没有可用 LLM 后端**时不报错退出,退回**确定性机械合并**(两边 brief 的机械并集,不去重、不判冲突),
  离线也能出一份统一上下文;标注清楚,装好模型后重跑可得语义合并。

> **有模型的那一步是非确定性的**(交给模型),同样输入两次结果可能不同。这是取舍。
> 它会在什么 case 上出问题(悄悄合错、不可复现、prompt 注入、丢了过期检测……)见
> [`风险分析.md`](风险分析.md) —— 用之前值得读一遍。

## 7. 换 LLM 后端

reconcile 用到的模型统一走 `src/llm.rs`,可插拔:

```sh
export AGIT_LLM=claude               # 默认,本机 claude -p
export AGIT_LLM=codex                # 本机 codex exec(已接,内部走 codex exec -o <file> - 取干净回复)
export AGIT_LLM_CMD="<从 stdin 读 prompt 的任意命令>"   # 覆盖一切,如 ollama
```

三个后端都不可用时,reconcile 退回**确定性机械合并**(见 §6),不硬性依赖模型。

**Codex 已是一等 runtime**(不再是桩):它的 session dump(`~/.codex/sessions/`,按 `cwd` 过滤本项目)
能 `agit -a sync codex`、`reconcile`、`convert`。解析在 `src/adapter/codex.rs`。

## 8. Hub

团队级托管 + 浏览,详见 [`hub.md`](hub.md)。要点:Hub 只做 **Registry + Sync + 只读渲染**,
**不合并**(合并在消费者本地 reconcile)。前端是内嵌的 React SPA(会话脊线 / provenance / permalink / revision diff)。
**push 需写 token**(`agit-hub token add … --write`,只存 sha256 摘要);`serve --private` 时读也要 token。

## 9. 安全与已知局限

- **`agit -a scan` / commit / push hook** 扫 session 里的密钥。dump 全会话 = agent cat 过的
  `.env`、连接串都在转录里,不扫就发出去。jsonl 按 JSON 解析、只扫**字符串值**,并按字段名和
  shape(uuid / sha / 时间戳 / 路径 / 内容寻址哈希)挡掉转录噪声,所以熵检测能开着 ——
  不匹配任何已知前缀的凭据也抓得到;但**拦不住一般性敏感内容**(商业机密、PII)。
- **误报的逃生口**(没有它,人只会去用 `--no-verify` 把整道闸门关掉):
  行内加 `agit:allow-secret` 豁免该行;或在被扫根目录放 `.agit-allow-secrets`,
  每行一条字面子串、或 `re:` 开头的正则。
- **reconcile 非确定性**、依赖外部模型、把不可信会话喂给 LLM(prompt 注入面)。
- **丢了旧模型的"证据→代码"过期检测**。
- 以上每一条的细节和缓解方向,见 [`风险分析.md`](风险分析.md)。

## 10. 完整工作流示例

```bash
# ── Alice ────────────────────────────────
cd payments && agit init
agit -a sync                                   # 镜像她这次会话
agit -a add -A && agit -a commit -m '查清用户模型'
# Hub 管理员先发一个写 token:agit-hub token add alice --write（token 只显示一次）
agit -a remote add origin http://alice:<token>@hub:8177/payments.git
agit -a push -u origin main

# ── 小林(第二天)─────────────────────────
cd payments
agit clone http://hub:8177/payments.git        # 拉团队 Agent Store
agit -a sync && agit -a add -A && agit -a commit -m '我这次会话'
agit -a fetch origin
agit -a reconcile origin/main                  # agent 合并:统一上下文进 CLAUDE.md,真冲突才问他
# 现在启动 claude,新会话自动带着两个人合起来的上下文
```

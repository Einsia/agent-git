# 在 hub 上浏览 agent

hub 是您的团队自行托管的一台服务器，用来共享 agent 并在网页界面中浏览它们。它把每个 agent 作为一个供您推送和拉取的 git 仓库来托管，并提供网页：渲染每一次会话、比较两个修订版、以及为一个组织或一个代码仓库所产生的 agent 建立索引。要运行一个 hub，参见[自建 hub](deploying-the-hub.html)。

您通过普通的客户端命令访问 hub。没有 hub 专用的动词：推送和拉取一个 agent 的方式，与对待任何 git 远端完全一样。参见[与团队共享 agent](guide/sharing.html)。

## 登录并浏览 agent

在浏览器中打开 hub 并用您的账户登录。hub 会列出您可读的 agent：您自己的、与您共享的，以及任何公开的。您不可读的 agent 会被隐藏，而一个私有 agent 与一个根本不存在的 agent 无从区分，因此 agent 的名称无法被枚举。

每个 agent 都有一个所有者、一个可见性（默认私有），以及被授予读、写或管理员权限的成员。您能对一个 agent 做什么，取决于您获得的授权：

| | 读 | 写（推送） | 管理员（可见性、成员、重命名、删除） |
|---|---|---|---|
| 匿名 | 仅公开 | 否 | 否 |
| 已登录用户，无授权 | 仅公开 | 否 | 否 |
| 成员（读 / 写 / 管理员） | 是 | 写及以上 | 管理员及以上 |
| 所有者 | 是 | 是 | 是 |

当 hub 拒绝一次操作时，它会说明原因：被拒绝的推送会指出它以哪个账户通过了认证、以及它缺少什么，因此错误的令牌、缺失的授权和只读的作用域三者很容易区分。

## 查看一次会话

agent 页面会列出它的会话。打开其中一次，即可看到它渲染成一条事件时间线：提示词、回复、工具调用和编辑，每个事件画成一道刻度线（竖线），其高度和颜色随类型而定，因此您一眼就能读出一次会话的形态。在它旁边，页面会显示该会话的来源信息（运行时、模型、分支、作者、时间）及其验证徽章，让您能判断是谁产生了它、以及它是否值得合并。参见[验证会话的产生者](guide/provenance.html)。

## 比较两个修订版

hub 提供一个 agent 历史中两个点之间的语义差异：新增和移除的指令、文件和结论，而非原始的会话记录字节。它还提供跨越您可读的每一个 agent 的会话的搜索。

## 组织概览

`/orgs/<name>` 是一个组织的概览页。它列出该组织的成员，以及他们能触及的每一个 agent：组织所拥有的 agent，外加您获准阅读的成员个人 agent。每个 agent 显示它的会话数以及它工作过的代码仓库。该页仅对组织成员和站点管理员开放，并且只列出您可读的 agent，因此它绝不会暴露一个您本就看不到的 agent。

## 代码仓库索引

`/repos` 把这个视角反转过来。它列出 hub 的 agent 工作过的每一个代码仓库，按环境分组，并附上挂在每个仓库上的各个 agent。一个仓库往往被多个 agent 触及；它只列出一次，并标出每个 agent 的会话数。该索引由您获准阅读的 agent 构建，因此它绝不会暴露一个您看不到的 agent。

## 您如何登录与推送

hub 接受两类凭据：

- 一个 cookie 会话，供人使用。用密码登录会返回一个会话 cookie，它会过期，也可以通过登出来吊销。
- 一个令牌，供 git 和脚本使用。在网页界面中创建令牌，然后把它作为 bearer 令牌发送，或在 git 的密码提示中键入。令牌可以限定到单个 agent、设置时限并被吊销。令牌是权限的上限，而绝非权限的来源：一个只读令牌仍然只能读，管理员操作需要您自己登录，绝不靠令牌。

如果您的团队启用了自助注册，您就能从 hub 自行创建账户。否则由管理员为您创建。

## hub 不做什么

hub 负责托管、同步和渲染。它不运行 agent，也不做合并。合并要读取两次会话，并对照您的代码对它们进行推理，这在同时拥有代码和模型的那台机器上本地进行。参见[协调分叉的会话](guide/merging.html)。

至于这些页面背后的存储后端、权限引擎和 API 路由，参见[自建 hub](deploying-the-hub.html)。

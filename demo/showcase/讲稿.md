# 大合集演示 · 主持人讲稿

> 每一步四件事：**① 敲什么 · ② 屏幕出现什么 · ③ 代表我们的什么能力 · ④ 可以照着念的话**。
> 屏幕输出取自真实运行；摘要（digest）等短哈希每次一致（seed 固定）。
>
> 开场先把环境准备好（**换新终端要重跑这两行**）：
> ```sh
> ./demo/showcase/setup.sh
> export PATH="/tmp/agit-demo/bin:$PATH"
> cd /tmp/agit-demo/showcase-alice
> ```
> 一句话主线：**Alice 干活 → 抽成带出处的 context → 发布到 Hub；Lin 一条命令复用、核实、合并、装回自己的 Claude Code。**

# 第一幕 · Alice：从会话到带出处的 context

## ① 两个库 —— context 和代码分开

```sh
agit init
agit-state
```

**屏幕出现：**
```
代码仓库 .gitignore 追加: .agit/
agit 已就绪。
  Environment : .../showcase-alice
  Agent Store : .../showcase-alice/.agit/agent

（agit-state：）
1. Environment —— 你的代码，原封不动      HEAD 7a891b9  .agit/ 被忽略？ 是
2. Agent Store（.agit/agent）              HEAD 10b95e3  fact: 0 条  merge driver: 已装
3. WorkspaceRevision                       （还没有。任一库 commit 后自动生成）
```

**代表的能力：** 版本化的对象从「代码」扩展到「代码 + Agent 的知识」。
**念白：** 「agit 在你的代码仓库旁边建了第二个 git 仓库 `.agit/agent`，专门装 agent 的上下文，
并自动加进 .gitignore——**context 和代码分开，互不污染**。一共就三个地方，没有隐藏数据库。」

## ② scope —— 一套 git 习惯，切两个库

```sh
agit status --short          # 代码仓库
agit -a status --short       # Agent Store（此刻是空的）
```

**屏幕出现：** 上面一条列出代码改动（`M .gitignore` 等）；下面一条空（context 还没东西）。

**代表的能力：** 零学习成本——你已有的 git 习惯直接迁移，`-a` 前缀切到 context 库。
**念白：** 「命令还是 git 那一套。不加前缀作用在代码，加 `-a` 作用在 context 库。
留个悬念：`agit -a commit` 和 `agit commit -a` 是两回事，第七步会踩到。」

## ③ 从一次 agent 会话抽取 context

```sh
agit -a import session.jsonl
cat .agit/agent/state/_evidence_pool.md
```

**屏幕出现：**
```
从 claude-code session 抽取 AgentState：
  目标      : 1 条 prompt
  证据池    : 2 条 file 证据已对齐基线，0 条跳过
              1 条命令（只记不跑）
  artifact  : 1 个

（证据池：）
- `file:models/user.ts:4 #a937b4a5`
- `file:services/order.ts:5-8 #7705f745`
- `cmd:grep -n user_id models/user.ts`
```

**代表的能力：** ContextManagement——把 agent「读过什么、跑过什么」确定性地提炼成可复用的原材料。
**念白：** 「这一步不调模型，纯解析：agent 真读过的文件成了 `file:` 证据、真跑过的命令成了 `cmd:` 证据，
而且**当场对齐到当前代码**算了摘要。这是结论的原材料。」

**（可选加演）让本机 claude 归纳成结论：**
```sh
agit -a import --summarize session.jsonl
```
> 念白：「加 `--summarize` 就调本机 claude 把证据池归纳成带出处的 fact。**关键**：模型只能引用
> 证据池里的东西，编造出处在构造上不可能。」（加演过就跳过④，两者取其一。）

## ④ 手写一条结论，证据指向真实代码

```sh
agit -a new api/user/id-field-name -e file:models/user.ts:4 -m '用户标识字段叫 user_id，不是 uid。'
```

**屏幕出现：**
```
新建 fact  state/facts/api/user/id-field-name.md
  tier: reversible
  证据: file:models/user.ts:4 #a937b4a5
```

**代表的能力：** 每条结论都**带证据出处**，没有证据的结论进不来。
**念白：** 「主题 `api/user/id-field-name` 就是文件路径；`#a937b4a5` 是采集当时那一行的摘要。
指一个不存在的行试试，它会直接拒绝——**没有出处的结论不入库**。」

## ⑤ 校验 + 出处链

```sh
agit -a verify
agit -a why api/user/id-field-name
```

**屏幕出现：**
```
FRESH  api/user/id-field-name  [FRESH] file:models/user.ts:4 #a937b4a5
                                 ↳ models/user.ts:4 → user_id: string;
1 条 fact，证据全部新鲜。

（why：）
结论  用户标识字段叫 user_id，不是 uid。
出处链  [FRESH] file:models/user.ts:4 #a937b4a5
```

**代表的能力：** 结论可追溯、可核实。
**念白：** 「verify 回到代码把每条证据重读、重算摘要、比对。why 给你结论的来龙去脉。
——记住这个 FRESH，第二幕要变。」

## ⑥ 密钥进不来

```sh
agit -a new db/pw -e file:.env:1 -m '连接串在这。'
```

**屏幕出现：**
```
agit: 证据无法采集: file:.env:1: .env 在 denylist 上（.env / 私钥 / 被 gitignore 的路径）。
agit 拒绝把它的内容抄进 context —— 这正是密钥泄漏的路径。
```

**代表的能力：** context 会被 push/pull，所以密钥有三道防线，这是第一道（采集时拒绝）。
**念白：** 「agent 常会 cat 到 .env。context 一旦分享出去，密码就发到每个人机器上。
所以 `.env`、私钥、被 gitignore 的路径**根本不许进 context**。」

## ⑦ 提交 context —— 注意 -a 的位置

```sh
agit -a add -A
agit -a commit -m 'alice：用户模型与延迟结论'
```

**屏幕出现：** `[main d26bf9f] alice：…  6 files changed`

**代表的能力：** 那个悬念——scope 开关只认紧跟 agit 的第一个 token。
**念白：** 「`agit -a commit` 提交的是 context。如果写成 `agit commit -a`，那个 `-a` 是 git 的参数、
提交的是**代码**。位置决定一切。」

## ⑧ 配对 + 校验 + 可移植引用

```sh
agit workspace
agit -a validate
agit -a portable
```

**屏幕出现：**
```
（workspace：）{ "agent_rev": "d26bf9f…", "env": { "head_commit": "7a891b9…",
                 "stash_tree": "c58d655…", "dirty": true }, "trigger": "agent:commit" }
（validate：） agit/v1-draft 校验：1 条 fact  ✓ AgentState 合法，无密钥
（portable：）{ "agent_state_ref": "d26bf9f…", "history_ref": "session", … }
```

**代表的能力：** JointVersionControl——把「context 版本」钉到「它所基于的代码版本」；
schema 校验；跨机器复用的引用。
**念白：** 「每次提交，agit 自动记一条配对：这份 context 是基于哪个代码提交得出的，
连没提交的工作树都进了 stash_tree。**结论永远不脱离代码基线**。」

## ⑨ 发布到团队 Hub

```sh
agit -a remote add origin http://localhost:8180/payments-api.git
agit -a push -u origin main
```

**屏幕出现：**
```
扫描 1 个文件，未发现密钥。
 * [new branch]      main -> main
```
**然后打开浏览器：** `http://localhost:8180/`

**代表的能力：** TeamExposure——Agent Store 就是 git 仓库，push 走真的 git 协议；发布前再扫一遍密钥。
**念白：** 「push 前 hook 又扫了一遍密钥（第三道防线）。现在团队打开这个网页，就能直接读 Alice 的
目标、带出处的结论、历史——**不用问她、不用翻聊天记录**。」

# 第二幕 · 证据会过期（全场最想让人记住的一点）

```sh
sed -i 's/  user_id: string;/  userId: string;/' models/user.ts
git commit -qam '重命名 user_id -> userId'
agit -a verify
```

**屏幕出现：**
```
STALE  api/user/id-field-name  [STALE] file:models/user.ts:4 #a937b4a5
                                 ↳ models/user.ts:4 已变更（a937b4a5 → 06208f05）
1 / 1 条 fact 的证据已失效或不可达。
```

**代表的能力：** 证据随代码自动失效——**这是三个竞品原理上做不到的事**。
**念白：** 「有人把 user_id 改成了 userId，没人记得改 context。**agit 自己发现了**——
那条结论变 STALE，退出码非零，可以直接挂 CI。Claude Code 的 rewind、Zed 的 checkpoint、
Shepherd 都做不到，因为它们记录的是状态，不记录指向源头的指针。」

# 第三幕 · 合并冲突用证据裁决

```sh
# 队友分支：据 2024 老文档
agit -a checkout -b teammate
agit -a new refund/status-field -e doc:docs/api-v1.md@2024-03-11 -m '退款状态字段叫 status。'
agit -a add -A && agit -a commit -m 'teammate：据 2024 文档'
# Alice 分支：据代码
agit -a checkout main
agit -a new refund/status-field -e file:services/refund.ts:8 -m '退款状态字段叫 state。'
agit -a add -A && agit -a commit -m 'alice：据代码'
# 合并
agit -a merge teammate
cat .agit/agent/state/facts/refund/status-field.md
```

**屏幕出现（冲突文件末尾）：**
```
CONFLICT (add/add): Merge conflict in state/facts/refund/status-field.md

# ─────────────── agit 证据裁决 ───────────────
# ours   [FRESH] file:services/refund.ts:8 —— 活代码
# theirs [STALE] doc:docs/api-v1.md@2024-03-11 —— 854 天前采集
# 建议采纳: ours  （依据合并时的证据状态，非模型判断）
#   agit -a resolve refund/status-field --take ours
```
```sh
agit -a resolve refund/status-field --take ours
```

**代表的能力：** SemanticMerge——冲突时**用证据裁决**，且确定性。
**念白：** 「两个 agent 对同一字段给了对立结论。merge driver **在合并那一刻重新校验双方证据**：
Alice 的是活代码（FRESH），队友的是两年前文档（STALE），建议采 Alice 的。
**同样输入跑一万次结果一样，模型不进裁决路径**；两边一样新时它会拒绝猜、交给人。」

# 第四幕 · Lin：一条命令复用，装回自己的 Claude Code

```sh
cd /tmp/agit-demo/showcase-lin
agit clone http://localhost:8180/payments-api.git
agit -a verify
```

**屏幕出现：**
```
已拉取 Agent Store ← http://localhost:8180/payments-api.git
agit 已就绪。

（verify：）FRESH  api/user/id-field-name  …  1 条 fact，证据全部新鲜。
```

**代表的能力：** ContextReuse——一条命令拿到团队 context，并**对自己的代码基线复验**。
**念白：** 「Lin 一条命令把整个 context 拉下来。verify 是对**他自己**的代码跑的——
他拿到的不是『Alice 说有个结论』，而是『Alice 说的，依据这几行代码，我这边刚复验过还成立』。」

```sh
agit -a export --to claude-code
sed -n '1,14p' CLAUDE.md
```

**屏幕出现：**
```
已写入 .../CLAUDE.md（受管区块）
  下一个 Claude Code 会话会自动加载它 —— 一开工就带着这份 context。

（CLAUDE.md：）
## 目标
- 用户标识字段叫什么？订单列表为什么慢？
## 已知事实（带证据）
- **api/user/id-field-name** — 用户标识字段叫 user_id，不是 uid。
  - 依据 `file:models/user.ts:4 #a937b4a5`
```

**代表的能力：** 真正落到 runtime——写进 `CLAUDE.md`，下个 Claude Code 会话自动带着这份 context。
**念白：** 「这就是『一条命令在自己 runtime 复用』：写进 CLAUDE.md 受管区块，幂等、不动你手写的内容。
Codex 也留了接口，拿到样本就能接。」

**（收尾亮点）Hub 也直接给 Claude Code 就绪的 context：**
```sh
curl -s http://localhost:8180/agent/payments-api/claude.md
```
> 念白：「一条 `curl … >> CLAUDE.md` 就能从 Hub 把团队 context 塞进任何仓库。」
